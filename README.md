# モルック大ちゃん

モルック競技の作戦・練習を支援するスマートフォンWebアプリ。

## 現在のフェーズ

**Phase 1.1**: ピン配置エディタ + 軌道ベースのヒューリスティック作戦提案 + ピン盤UX改善

完了済み:
- Phase 0: ルール解説、スコア記録（2人対戦）、ローカル保存
- Phase 1: ピン配置の可視化と編集、倒しやすさのヒューリスティック評価、作戦提案（単本・複数本）
- Phase 1.1: スコア固定表示、履歴2列化、ピン盤ズーム、ピン盤の表示/非表示切替、各ピンに倒しやすさ%表示、軌道ベースに改良した単本ヒューリスティック

今後の予定:
- Phase 2: 作戦提示UI改善、勝ち筋の複数候補比較、複数人対戦
- Phase 3: MDP+価値反復による最適ポリシー組込（AI推奨モード）

## 技術スタック

- Vite + React + TypeScript
- Tailwind CSS
- Zustand（状態管理 + localStorage 永続化）
- SVG（ピン盤の描画と pointer events によるドラッグ／タップ）

## 起動方法

```bash
cd molkky-daichan
npm install
npm run dev
```

開発サーバーは `http://localhost:5173/` で起動します。同一LAN上のスマートフォンからは `http://<PCのIP>:5173/` でアクセス可能。

## ビルド

```bash
npm run build
npm run preview
```

ビルド成果物は `dist/` に出力されます。

## 機能

### 試合タブ
- ✅ プレイヤースコアの表示（現在ターン強調、残り点数、連続ミス可視化）
- ✅ ピン配置エディタ（SVGベース）
  - タップでピンを倒す／立てる
  - ドラッグでピン位置を調整
  - 「全ピンを立て直す」「初期配置に戻す」ボタン
- ✅ 投擲確定ボタン
  - 倒れたピン数で得点を自動判定（0本=ミス、1本=その番号、2本以上=本数）
  - 確定すると全ピンが立った状態に戻る
- ✅ 作戦提案
  - 立っているピンに対する単本狙い・複数本まとめ狙いを評価
  - 倒しやすさ × 期待利得 で並び替え
  - タップでピン盤上にハイライト表示
- ✅ 手動入力（番号パッド）も併用可能
- ✅ 1手戻し、新ゲーム、投擲履歴

### ルールタブ
- ✅ モルックの基本ルール解説

### 設定タブ
- ✅ プレイヤー名編集
- ✅ 連続ミス上限の調整（2/3/4回）

### その他
- ✅ ローカルストレージへの自動保存
- ✅ スマホ片手操作前提のレイアウト

## ヒューリスティックの内訳

`src/domain/heuristic.ts` で実装。

- **単本狙いの倒しやすさ** = 距離ファクター × (1 − 軌道リスク)
  - 距離ファクター: 投擲位置から対象ピンまでの距離（近いほど高評価）
  - 軌道リスク: 投擲位置 → 対象ピンの直線（軌道）に他のピンが絡む度合い
    - **軌道方向（前後）と垂直方向（横ずれ）に分解して評価**
    - 対象の真後ろ近くに別ピン → 同時倒しリスクで強く減点
    - 対象の手前に別ピン → ブロッキングリスクで中程度減点
    - 横にずれているピン → 干渉なし
- **複数本狙いの倒しやすさ** = 距離ファクター × 密集度ファクター × サイズ減衰
  - 密集度: クラスタ内のピンが近いほど高評価
  - サイズ減衰: クラスタが大きいほど指数的に低評価
- **期待利得** = 倒しやすさ × 効用
  - 50点ぴったり = +100、超過（25戻り）= 強い負の効用、それ以外 = 利得

各立っているピンの「倒しやすさ%」はピン盤上に直接表示される（70%以上=緑、40〜70%=橙、40%未満=赤）。

これらのパラメータ・式は Phase 2 以降で実利用ベースに調整していく。

## Phase 3 への布石

ドメイン層は純関数で構成しているため、Phase 3 で Python 側の MDP 遷移関数の仕様としてそのまま流用可能。
- `applyAction(state, action)` → 状態遷移
- `generateStrategies(...)` → 行動候補列挙
- `easeSinglePin / easeCluster` → 命中確率の近似
